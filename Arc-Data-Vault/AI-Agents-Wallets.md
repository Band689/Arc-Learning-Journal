## 🇺🇸 English Version

AI Agents Are Getting Wallets. That Might Be a Bigger Deal Than I First Thought.

One comment from Circle's latest earnings call stuck with me.

Jeremy Allaire said the future economy won't just be powered by people—it'll increasingly be driven by AI agents making decisions and completing tasks on their own.

At first, that sounded like another big vision statement.

But the more I thought about it, the more practical it started to feel.

If an AI agent is expected to book cloud compute, pay for APIs, subscribe to data feeds, or interact with other software autonomously, someone—or something—has to handle the payments. Stopping every few seconds to ask a human to approve a credit card transaction obviously doesn't scale.

That's where I think Circle's Agent Stack starts to make sense.

Instead of treating AI like another app, they're giving it financial primitives: wallets, programmable payments, and a way to discover and pay for services using USDC.

The feature that caught my attention wasn't actually the wallets.

It was the nanopayments.

A lot of AI interactions are tiny. Paying fractions of a cent for an API request or a few seconds of GPU time simply doesn't work well with traditional payment systems. If those payments can be authorized instantly and settled efficiently, entirely new business models become possible.

Circle also released starter kits for frameworks like LangChain, Claude Agent SDK, and the OpenAI Agents SDK, which tells me they're trying to make this easy for developers instead of expecting everyone to build payment infrastructure from scratch.

Of course, whether developers actually adopt it is a different question.

That's why I find examples like SumPlus and NodersPay more interesting than the product announcement itself. They're early, but at least they show what this could look like outside a demo.

I also think it's worth separating the long-term vision from what's happening today.

We're probably not going to wake up next year to billions of AI agents paying each other. But I do think software is becoming increasingly autonomous, and if that trend continues, payments designed for humans may eventually become a bottleneck.

Maybe that's the real bet Circle is making with Arc.

Not that AI needs another blockchain—but that autonomous software will eventually need financial infrastructure built specifically for machines.

Curious how everyone else sees it.

Do you think AI-native payment rails become essential over the next few years, or are we still much earlier than the market believes?

## 🇨🇳 中文版本

AI Agent 开始拥有钱包了。这可能比我最初想象的要重要得多。

在 Circle 最近的财报电话会上，有一句话让我印象深刻。

Jeremy Allaire 提到：未来的经济形态不会只由人类驱动，AI 智能体（AI Agents）自主决策、独立完成任务的比例会越来越高。

刚听到时，我觉得这听起来又是一个宏大的愿景蓝图。

但越仔细想，越觉得这事其实非常现实。

如果我们要让一个 AI Agent 自主去预订云算力、购买 API 调用、订阅数据源，或者与其他软件进行交互，那么总得有某种机制来处理支付。如果每隔几秒钟就要暂停下来，等待人类去批准一次信用卡刷卡，这种模式显然是无法规模化的。

正因如此，我认为 Circle 推出的 Agent Stack 开始展现出它的合理性。

他们没有把 AI 当作普通的应用程序来看待，而是赋予了它基础的金融原语：钱包、可编程支付，以及一种通过 USDC 来发现服务并进行支付的机制。

不过，最吸引我注意的其实不是钱包本身，而是**纳米级微支付（Nanopayments）**。

AI 之间的很多交互是非常微小的。为了单次 API 请求或几秒钟的 GPU 算力去支付几分甚至几毫美分，在传统支付系统下根本无法实现。如果这些微支付能够被瞬间授权并高效结算，全新的商业模式将会被彻底打开。

此外，Circle 还针对 LangChain、Claude Agent SDK 以及 OpenAI Agents SDK 等主流框架推出了开发入门套件（Starter Kits）。这表明他们正在试图降低开发者的门槛，而不是让大家从零去搭建一套支付基础设施。

当然，开发者最终是否会大规模采用，依然是一个未知数。

这也是为什么比起产品发布公告本身，我对像 SumPlus 和 NodersPay 这样的真实案例更感兴趣。虽然它们还处于非常早期，但至少展示了这种模式走出 Demo 后的实际模样。

同时，我也认为有必要将长期愿景与当下的现实区分开来。

我们大概率不会在明年一睁眼，就看到几十亿个 AI Agent 在互相转账。但我坚信软件正变得越来越自主化，如果这个趋势持续下去，**为人类设计的传统支付管道最终会成为机器自主化的瓶颈**。

或许，这才是 Circle 布局 Arc 的真正下注所在：

**并不是 AI 需要一条区块链，而是自主化的软件最终将需要一套专门为机器构建的金融基础设施。**

很想听听大家的看法：你认为“AI 原生”的支付管道会在未来几年内成为刚需，还是说我们目前依然处于比市场预期要早得多的阶段？
